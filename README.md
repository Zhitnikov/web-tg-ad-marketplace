# web-tg-ad-marketplace

## Описание проекта

AdChannel Marketplace — это веб-приложение, представляющее собой маркетплейс для рекламы в Telegram-каналах. Компании могут выставлять запросы на размещение рекламы, а владельцы Telegram-каналов — забирать эти заказы, публиковать рекламу у себя и получать оплату. Приложение облегчает взаимодействие между рекламодателями и владельцами каналов, автоматизируя процесс подбора, оплаты и отслеживания.

Проект реализован как монопо-репозиторий с разделением на фронтенд (React + TS + Vite) и бэкенд (C# + .NET). Для генерации рекомендаций и анализа используется интеграция с LLM через Ollama, что позволяет автоматически создавать посты для рекламы. Решение соответствует ограничениям: используются только открытые библиотеки и бесплатные LLM-модели без подписок.

Целевая аудитория — компании (рекламодатели) и владельцы Telegram-каналов. Приложение решает проблемы поиска подходящих каналов, согласования условий и мониторинга эффективности рекламы.

## Функциональность

- **Для рекламодателей**: Создание и размещение запросов на рекламу, поиск подходящих каналов, оплата заказов, отслеживание результатов.
- **Для владельцев каналов**: Просмотр доступных заказов, принятие заказов, публикация рекламы, получение выплат.
- **Интеграция LLM**: Генерация рекомендаций по тексту рекламы, анализу аудитории каналов и оптимизации кампаний.
- **Дополнительные возможности**: Аутентификация, уведомления, управление заказами, кошелек для транзакций, интеграция с внешними API (например, погода для контекстной рекламы).
- **Пользовательский интерфейс**: Веб-платформа с интуитивным дизайном, оптимизирована для мобильных устройств.
- **Масштабируемость**: Архитектура позволяет добавлять новые функции и интеграции (например, с банковскими системами или Telegram API).

## Архитектура

- **Фронтенд**: React с Vite для быстрой разработки и сборки. Компоненты для аутентификации, создания объявлений, заказов и кошелька. Нет Docker-образа; запуск через npm.
- **Бэкенд**: C# на .NET с Entity Framework для миграций и базы данных (PostgreSQL). Обработка API-запросов, интеграция с LLM через Ollama.
- **База данных**: PostgreSQL в Docker-контейнере.
- **LLM**: Ollama с моделью lakomoor/vikhr-llama-3.2-1b-instruct:1b для локального inference. Интеграция обеспечивает точные, контекстные рекомендации по рекламе.
- **Обоснование**: Выбор .NET для бэка — стабильность и производительность; React для фронта — удобство UI. Ollama выбрана за открытость и локальный запуск, без внешних API. Архитектура модульная для горизонтального масштабирования и интеграций.

## Требования

- Docker и Docker Compose для базы и Ollama.
- .NET SDK для бэка и миграций.
- Node.js (v18+) и npm для фронта.
- PostgreSQL клиент для подключения (опционально).

## Установка и запуск

### 1. Клонирование репозитория
- `git clone [https://github.com/](https://github.com/Zhitnikov/web-tg-ad-marketplace.git)`
- `cd web-tg-ad-marketplace`

### 2. Настройка бэкенда
- Перейдите в папку `backend`.
- Установите зависимости: `dotnet restore`.
- Создайте миграцию (если нужно): `dotnet ef migrations add InitialMigration --output-dir infrastructure/Persistence/Migrations`.
- Обновите базу: `dotnet ef database update`.

Для отката: `dotnet ef database update PreviousMigrationName`.

### 3. База данных (PostgreSQL)
Запустите Docker-контейнер:
- `docker compose up -d --build daemon`

- Подключение: `jdbc:postgresql://localhost:1337/adchannel?user=postgres&password=postgres`.
- Просмотр логов: `docker-compose logs`.
- Остановка: `docker-compose down`.

### 4. LLM (Ollama)
- Запустите Ollama контейнер (если не запущен): `docker run -d -v ollama:/root/.ollama -p 11434:11434 --name ollama ollama/ollama`.
- Установите модель: `docker exec -it ollama ollama run lakomoor/vikhr-llama-3.2-1b-instruct:1b`.

### 5. Фронтенд
- Перейдите в папку `frontend`.
- Установите зависимости: `npm install`.
- Запуск в разработке: `npm run dev`.
- Сборка для продакшена: `npm run build`. Результат в `dist/`.

### 6. Полный запуск
- Запустите базу и Ollama через Docker.
- Запустите бэкенд: `dotnet run` в `backend`.
- Запустите фронтенд: `npm run dev` в `frontend`.
- Доступ: Откройте браузер на `http://localhost:5173` (или порт из Vite).

## Развертывание
- **Локально**: Как выше.
- **На сервере**: Соберите Docker-образ для бэка (добавьте Dockerfile в `backend`), разверните на Vercel/Render. Фронт — на Netlify/Vercel как статику.
